# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
내 맘대로 폴더링 맵 (나만의 커스텀 테마 맛집/ 카페 지도)
<br>


## 2. 시나리오 (5~10줄)

* 이 프로그램은 사용자가 직접 방문하거나 가고 싶은 음식점 및 카페만을 검색하여, 자신만의 기준(0~5점, 1점 단위)으로 별점을 매기고 짧은 한줄평을 기록해 다중 폴더에 담아 관리하는 개인 맞춤형 지도 시스템입니다. 사용자가 폴더별로 지정한 고유 이모티콘(마커)이 지도 위에 직관적으로 표시되어, 나만의 테마 지도를 한눈에 파악할 수 있습니다. 또한, 하나의 맛집을 여러 폴더에 중복으로 넣을 수 있는 유연한 관리 시스템과 폴더 내 검색 기능을 지원하여, 텍스트 형태가 아닌 시각적으로 완성도 높은 나만의 커스텀 맛집 지도를 구축할 수 있도록 기획되었습니다.
<br>


## 3. 예상 기능 및 메뉴 (최소 5개)

1. 맛집/카페 전용 검색 및 선택: API 카테고리 필터링을 통해 오직 음식점과 카페만 검색되며, 유사한 이름의 가게 목록 중 원하는 곳을 정확히 클릭하여 선택.
2. 직관적인 별점 평가 및 한줄평 기록: 0~5점 사이에서 1점 단위로 평가. (★, ☆)하는 별점 시스템과 더불어, 해당 맛집에 대한 나만의 방문 후기나 메모를 한줄평으로 자유롭게 기록하고 저장.
3. 유연한 다중 폴더링 및 검색: 평가한 식당을 여러 테마 폴더에 동시 저장 가능. 폴더 내에서 가게 이름을 다시 검색하거나, 전체 맛집을 특정 폴더로 분류하는 양방향 탐색 지원
4. 폴더별 커스텀 이모티콘 맵핑: 각 폴더에 부여된 고유 이모티콘이 지도 API 위에서 직접 커스텀 마커로 렌더링되어 색다른 시각적 경험 제공.
0. 프로그램 종료

### **🤖 AI 파트너십 과정**

 1. **내용 1 : 기획 구체화 및 방향성 설정**

    * **적용 내용:** 인생 영화 아카이브에 걸맞는 감상 횟수 변수를 추천해줌. 이를 정수형으로 설정하여 텍스트(str), 별점(float)과 함게 과제의 필수 자료형 조건을 충족하도록 기획함.
      
 2. **내용 2 : 추가 기능에 대한 기술적 실현 가능성 검토 및 아이디어 제안**
    * **프롬프트 요약:** "지도에 선택한 맛집들이 폴더별 이모티콘으로 나오게 하는 게 가능할까? 무조건 음식점/카페만 되게 해주고, 불가능한 내용이 있으면 바로 알려줘."
    * **적용 내용:** 
    1. 카카오맵 API의 커스텀 마커 기능을 활용하면 폴더별 이모티콘 지도를 100% 구현할 수 있음을 확인하여 핵심 기능으로 채택함.
    2. 카카오 장소 검색 시 카테고리 코드(FD6, CE7)를 고정해 음식점/카페만 노출되도록 필터링 로직을 추가함.
    3. 초기에 논의된 '실시간 공유 및 댓글' 기능은 파이썬 기초 과제 범위를 초과하므로 기획에서 과감히 제외하여 실현 가능한 목표로 범위를 조정함.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
(예시)
  * 이번 과제는 용사 등록 및 정밀 진단 시스템 구현으로, 사용자로부터 데이터를 입력받아 리스트에 저장하고, 가중치 연산을 통해 결과값을 도출하는 시스템을 구축하는 것입니다. 특히 `if-elif` 조건문과 `and/or` 논리 연산자를 활용하여 상황에 맞는 등급과 특별 칭호를 부여하는 지능형 로직을 완성해야 합니다.
  * input()으로 용사의 이름, 체력, 마력, 공격력을 입력받아 리스트에 저장.
  * 가중치 공식을 적용하여 종합 전투력 산출: (체력 * 0.7) + (마력 * 0.5) + (공격력 * 1.8).
  * if-elif-else를 사용하여 S등급부터 F등급까지 5단계 등급 판정 로직 추가.
  * and 연산자를 활용하여 '전설의 대마법사' 등 특별 칭호 부여 기능 구현.
    
### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1: 길드 매니저 컨셉 및 로직 도출**
    * **프롬프트 요약:** "내가 길드 매니저가 되어 용사를 관리하는 프로그램을 만들 거야. 1학년 수준에서 리스트와 조건문을 활용해 등급을 매기는 재미있는 로직을 짜줘."
    * **적용 내용:** AI가 제안한 '가중치 전투력 산출' 방식과 'S등급이면서 마력이 높으면 대마법사 칭호 부여'라는 복합 조건 아이디어를 실제 코드에 반영함.

2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수/딕셔너리)에 맞춰, '용사 길드 관리자' 컨셉의 프로그램이 어떻게 발전할 수 있을지 4단계 기능 확장 로드맵을 제안해줘."
    * **적용 내용:** 단순 입출력을 넘어, **'기초 정보 입력 및 리스트 저장(1차) → 전투 등급 판독기 구현(2차) → 무한 메뉴 시스템 구축(3차) → 딕셔너리를 활용한 다중 캐릭터 관리(4차)'**로 이어지는 논리적인 확장 로드맵을 확립함.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 숫자를 입력했는데 계산이 안 되는 현상 (TypeError)**
    * **원인:** `input()`으로 받은 데이터는 기본적으로 '글자'라서 숫자와 곱셈 연산이 불가능했음.
    * **해결:** `int()`와 `float()` 함수를 사용하여 입력값을 숫자로 바꾸는 '형변환' 마법을 부려 해결함.
2. **문제 2: 조건문 뒤에 자꾸 생기는 빨간 줄 (SyntaxError)**
    * **원인:** `if`와 `elif` 조건식 끝에 마법의 기호인 `:`(콜론)을 자꾸 빠뜨려서 발생함.
    * **해결:** 파이썬의 모든 조건문과 반복문 끝에는 항상 땡땡(`:`)이 들어가야 한다는 규칙을 학습하고 수정함.
