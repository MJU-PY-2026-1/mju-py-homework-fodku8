# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
내 맘대로 폴더링 맵 (나만의 커스텀 테마 맛집/ 카페 지도)
<br>


## 2. 시나리오 (5~10줄)

* 이 프로그램은 사용자가 직접 방문하거나 가고 싶은 음식점 및 카페만을 검색하여, 자신만의 기준(0~5점, 1점 단위)으로 별점을 매기고 짧은 한줄평을 기록해 다중 폴더에 담아 관리하는 개인 맞춤형 지도 시스템입니다. 사용자가 폴더별로 지정한 고유 이모티콘(마커)이 지도 위에 직관적으로 표시되어, 나만의 테마 지도를 한눈에 파악할 수 있습니다. 또한, 하나의 맛집을 여러 폴더에 중복으로 넣을 수 있는 유연한 관리 시스템과 폴더 내 검색 기능을 지원하여, 텍스트 형태가 아닌 시각적으로 완성도 높은 나만의 커스텀 맛집 지도를 구축할 수 있도록 기획되었습니다.
<br>


## 3. 예상 기능 및 메뉴 (최소 5개)

1. 맛집/카페 전용 검색 및 선택: API 카테고리 필터링을 통해 오직 음식점과 카페만 검색되며, 유사한 이름의 가게 목록 중 원하는 곳을 정확히 클릭하여 선택.
2. 직관적인 별점 평가 및 한줄평 기록: 0~5점 사이에서 1점 단위로 평가. (★, ☆)하는 별점 시스템과 더불어, 해당 맛집에 대한 나만의 방문 후기나 메모를 한줄평으로 자유롭게 기록하고 저장.
3. 유연한 다중 폴더링 및 검색: 평가한 식당을 여러 테마 폴더에 동시 저장 가능. 폴더 내에서 가게 이름을 다시 검색하거나, 전체 맛집을 특정 폴더로 분류하는 양방향 탐색 지원
4. 폴더별 커스텀 이모티콘 맵핑: 각 폴더에 부여된 고유 이모티콘이 지도 API 위에서 직접 커스텀 마커로 렌더링되어 색다른 시각적 경험 제공.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**

이번 과제는 1주차에서 기획한 '내 맘대로 폴더링 맵'의 기능인 식당 정보 입력 및 커스텀 별점 평가 시스템을 파이썬의 입출력, 리스트, 조건문을 활용해 구현하는 것입니다. 반복문(for/while)은 완전히 배제하였으며, 사용자로부터 식당 이름, 카테고리, 별점, 한줄평, 그리고 폴더명을 입력받아 하나의 리스트(List)에 저장합니다. 특히 입력된 데이터를 바탕으로 `if-elif-else` 다중 조건문을 통해 만족도 등급을 나누고, `match-case` 문법을 활용하여 사용자가 지정한 폴더명에 맞는 고유 이모티콘 마커를 부여하는 지능형 분기 로직을 완성했습니다.
* `input()`으로 식당 정보를 입력받고 숫자로 필요한 값은 `int()`로 형변환.
* 식당 카테고리가 '음식점(FD6)' 또는 '카페(CE7)'인지 확인하는 중첩 if문 구조 적용.
* `match-case`를 사용하여 '데이트', '혼밥' 등 사용자가 입력한 폴더명에 따라 지도 렌더링용 이모티콘(💖, 🎧 등)을 배정.
* 복합 논리 연산자(`and`, `or`, `==`)를 활용해 만점 평가와 특정 폴더가 결합된 경우 '✨ 소개팅 성공 확률 100% ✨'와 같은 특별 칭호 부여.
    
### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1: 폴더별 고유 이모티콘 마커 지정을 위한 로직 도출**
    * **프롬프트 요약:** "사용자가 '혼밥'이나 '데이트' 같은 폴더명을 입력하면, 그에 맞는 이모티콘을 지도 마커처럼 달아주고 싶은데 어떤 문법을 써야 할까?"
    * **적용 내용:** C언어의 `switch-case`와 대응되는 파이썬 3.10 버전 이상의 `match-case` 구문을 도입하는 아이디어를 AI로부터 제안받았습니다. 이를 통해 if문을 길게 나열하는 대신, 폴더명 패턴에 따라 직관적이고 가독성 높게 이모티콘을 맵핑하는 코드를 구현했습니다.

2. **내용 2: 음식점/카페 한정 필터링을 위한 중첩 조건문 설계**
    * **프롬프트 요약:** "지도에 무조건 음식점(FD6)과 카페(CE7)만 들어가게 하고 싶어. 그리고 그게 확인된 다음에만 별점 입력을 받으려면 어떻게 해?"
    * **적용 내용:** 입력의 가장 첫 단계에서 `if category == "FD6" or category == "CE7":` 라는 복합 조건식을 사용하여 카테고리의 유효성을 1차적으로 검증했습니다. 그리고 조건이 참일 때만 내부 들여쓰기 공간(중첩 구조)에서 별점 평가 및 폴더링 로직이 실행되도록 설계하여 잘못된 데이터 입력 시 즉각 종료되게 만들었습니다.
    
### **🛠️ Troubleshooting & 기술 회고:**

1. **문제 1: 입력한 별점 숫자로 등급을 계산하려는데 발생한 에러 (TypeError)**
    * **원인:** `input()` 함수로 입력받은 별점(예: 5)이 겉보기엔 숫자라도 파이썬 내부에서는 문자열(str)로 취급되어, 크기 비교 연산(>= 등)을 할 때 오류가 발생했습니다.
    * **해결:** `int()` 함수를 사용하여 `rating = int(rating_str)`로 변수 타입을 강제로 정수형으로 바꾸어주는 '형변환'을 거친 후 조건문을 실행해 말끔하게 해결했습니다.
    
2. **문제 2: 다중 조건문의 복합 연산 시 의도치 않은 결과 발생 (논리 오류)**
    * **원인:** `if category == "FD6" or category == "CE7" and rating == 5:` 와 같이 `and`와 `or`를 섞어 쓸 때, 연산자 우선순위를 고려하지 않아 원치 않은 상황에도 칭호가 부여되었습니다.
    * **해결:** 조건식을 `if (category == "FD6" or category == "CE7") and rating == 5:` 와 같이 괄호 `()`를 사용하여 평가 단위를 명확하게 묶어줌으로써 복합 논리식을 의도한 대로 동작시켰습니다.
